# The Twelve‑Factor App — описание WARC‑архива

- **Сайт:** [The Twelve‑Factor App](https://www.12factor.net/)
- **WARC‑файл:** `12factor.warc.gz` (ссылка или путь)
- **Дата и время сбора:** ноябрь 2025
- **Размер архива:** ≈ 1.16 МБ

## 1. Краткая характеристика сайта

«The Twelve‑Factor App» — это методология для создания современного программного обеспечения как сервиса. Она описывает двенадцать принципов, которые помогают разработчикам строить переносимые и масштабируемые приложения: управление зависимостями, хранение конфигурации в окружении, разделение стадий сборки и запуска и т. д.. Документ является коротким манифестом с разделами, посвящёнными каждому фактору, и доступен на разных языках.

## 2. Параметры сбора архива

Для сбора архива использовалась утилита `wpull` со следующими параметрами (пример):

```bash
wpull https://12factor.net/ \
  --warc-file=12factor \
  --recursive --level=1 \
  --page-requisites \
  --no-parent \
  --no-robots
```

### 3.1. Краткий разбор отчёта

Заголовки HTTP корректно задают тип содержимого (Content-Type: text/html; charset=utf-8) и сжатие (Content-Encoding: gzip), что облегчает работу ботов.
Есть файл sitemap https://12factor.net/sitemap.xml, упрощающий обход сайта.
Используются только локальные изображения (2 файла) и локальные таблицы стилей (2 CSS), что повышает связанность архива.
Часть JavaScript загружается с внешнего хоста (use.typekit.com), а также есть inline-скрипты — это усложняет полное воспроизведение при зависимостях от сторонних сервисов.
В отчёте зафиксированы ошибки в CSS и 2 ошибки валидности HTML (некорректные значения атрибута hreflang для pt_br и zh_cn), что снижает показатель соответствия стандартам.
Найдено 48 валидных и 1 некорректная ссылка; битая ссылка иллюстрирует типичную проблему link rot.
В HTTP-ответах отсутствуют явные заголовки кэширования, что снижает эффективность повторного обхода, но не критично для разового учебного захвата.


## 4. Метаданные WARC (metawarc)

Метаданные по объектам архива извлечены командой:

```bash
metawarc metadata --output metadata/12factor_meta.jsonl warcs/12factor.warc.gz
```
В файле 12factor_meta.jsonl содержатся 7 объектов. Распределение по MIME-типам (по полю mime):

    image/png — 2 объекта (например, privacy.png);

    image/webp — 3 объекта (например, nav-customers-l3-mobile.jpg, реально отдаётся как WebP);

    text/html — 1 объект (marquee-legal-background-desktop.jpg, но с MIME text/html);

    None — 1 объект (отсутствует явный MIME в метаданных).

    По расширениям файлов (поле ext):

    jpg — 4 объекта;

    png — 3 объекта.


## 5. Наблюдения и выводы

Структура сайта относительно простая, основной контент — текстовые HTML-страницы плюс несколько изображений и стилей.
ArchiveReady показывает, что при общем уровне архивируемости 70 % сайт всё же страдает от:
    небольшого числа невалидных ссылок;
    ошибок валидации HTML и CSS;
    зависимости от внешнего JavaScript.

По метаданным WARC видно, что:
    используются несколько форматов изображений (PNG, WebP), причём часть «маскируется» под .jpg по имени;
    один и тот же файл может фигурировать как «картинка по имени», но с MIME text/html (фактически HTML-страница).
