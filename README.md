# Коллекция веб-архивов: документация по современным DevOps-инструментам

Этот репозиторий содержит цифровую коллекцию веб-архивов, созданную с использованием инструмента `wpull` для загрузки сайтов, библиотеки `metawarc` для анализа WARC-метаданных и сервиса `ArchiveReady` для оценки архивируемости по метрикам CLEAR.

Тематика коллекции — **официальная документация и учебные разделы по современным инструментам разработки и эксплуатации**:

- методология **Twelve-Factor App**,
- туториал **Kubernetes Basics**,
- вводный раздел документации **Prometheus**,
- учебный курс **Docker Get Started**,
- официальная документация **FastAPI**.

---

## Краткое описание коллекции

Цель работы — собрать и описать небольшую, но хорошо структурированную коллекцию веб-архивов, отражающую экосистему облачной разработки и DevOps:

- как проектируются и описываются 12-факторные приложения;
- как организована обучающая документация по Kubernetes, Prometheus, Docker и FastAPI;
- насколько такие сайты пригодны для долгосрочного веб-архивирования (сколько там внешних ресурсов, картинок, скриптов, ошибок в верстке и т. д.).

Содержимое коллекции можно использовать:

- в учебных заданиях по курсу «Цифровые архивы»;
- как демонстрационный набор для анализа WARC-метаданных (`metawarc`);
- как «песочницу» для экспериментов с CLEAR-метриками в `ArchiveReady`.

---

## Метод и технологии

В работе использовались несколько инструментов.

1. **`wpull`** — основная утилита для загрузки веб-сайтов в формате WARC.  
   Использовались параметризованные вызовы с ограничением глубины (`--level=1`), захватом сопутствующих ресурсов (`--page-requisites`) и отключением `robots.txt` в учебных целях (`--no-robots`).

2. **`ReplayWeb.page`** — для проверки воспроизведения полученных архивов в браузере.

3. **`ArchiveReady`** — онлайн-сервис оценки архивируемости сайта (поддержка метрик CLEAR: Cohesion, Accessibility, Metadata, Standards Compliance).  
   Для каждого базового URL (`12factor.net`, Kubernetes Basics, Prometheus overview, Docker Get Started, FastAPI) запрашивались отчёты и фиксировались ключевые показатели.

4. **`metawarc`** — Python-библиотека для анализа WARC-архивов:
   - команда `metawarc metadata` использовалась для выгрузки метаданных в файлы `*_meta.jsonl`;
   - далее на их основе строились агрегированные сводки по MIME-типам и расширениям файлов.

## Общая статистика коллекции

На основании агрегированных таблиц `stats_exts_all.txt` и `stats_mimes_all.txt` можно охарактеризовать коллекцию следующим образом:

- Всего проанализировано **порядка 1100 объектов** без расширения (WARC-записей), суммарный объём — около **413 МБ**.00
- Среди файлов с расширением присутствуют:
  - `html` — страницы HTML-документации;
  - `css` — таблицы стилей;
  - `js` — скрипты;
  - `png`, `jpg`, `webp`, `svg` — изображения и схемы;
  - `woff2` — шрифты;
  - несколько служебных и редких расширений (`json`, `epub`, `one`, `run`, `webmanifest`) для отдельных ресурсов.

По MIME-типам преобладает веб-контент:

- **HTML-страницы** (`text/html` с разными кодировками) составляют подавляющее большинство ответов — более 1000 записей, что отражает документационный характер коллекции.  
- Есть заметная доля **JavaScript-ресурсов** (`application/javascript`, `text/javascript*`) и **CSS-файлов** (`text/css*`), обеспечивающих оформление и клиентскую логику сайтов.  
- Графика представлена форматами `image/png`, `image/svg+xml`, `image/webp`, всего несколько десятков файлов, в основном диаграммы и логотипы.  
- Шрифты описаны как `font/woff2`.

Более подробные цифры по расширениям и MIME-типам можно найти непосредственно в файлах:

- `stats_exts_all.txt` — сводка по расширениям;
- `stats_mimes_all.txt` — сводка по MIME-типам.

## Структура репозитория

Структура:

```text
archives-collection/
  README.md                   
  warcs
  metadata
  sites/
    12factor.md                # описание архива 12factor.net
    kubernetes-basics.md       # описание туториала Kubernetes Basics
    prometheus-docs.md         # описание Prometheus Introduction / Overview
    docker-get-started.md      # описание Docker Docs
    fastapi.md                 # описание архива документации FastAPI
````

### Файлы в `sites/`

Каждый файл в папке `sites/` следует единому шаблону:

* краткая характеристика сайта / раздела;
* параметры запуска `wpull` (основные флаги, глубина обхода);
* выдержки из отчёта `ArchiveReady` по метрикам CLEAR;
* описание выводов на основе метаданных `metawarc` (распределение по MIME-типам, изображений, внешних ресурсов);
* наблюдения и краткие выводы (особенности сайтa, проблемы архивирования, качество контента и верстки).

Это делает коллекцию однородной и облегчает сравнение разных ресурсов между собой.

## Как воспроизвести анализ

Ниже — пример минимального сценария для добавления нового сайта в коллекцию.

1. **Сохранить WARC-архив**

   ```bash
   wpull https://example.org/ \
     --warc-file=example-org \
     --recursive --level=1 \
     --page-requisites \
     --no-parent \
     --no-robots
   ```

2. **Извлечь метаданные WARC**

   ```bash
   metawarc metadata \
     --output metadata/example-org_meta.jsonl \
     warcs/example-org.warc.gz
   ```

3. **Обновить сводные статистики**

   * добавить `example-org_meta.jsonl` в общий расчёт `stats_mimes_all.txt` и `stats_exts_all.txt` (через `metawarc stats` или отдельный скрипт);
   * описать новый сайт в `sites/example-org.md` по существующему шаблону.

4. **Проверить сайт в ArchiveReady**

   * запустить анализ базового URL;
   * сохранить ключевые цифры и выводы в соответствующий `*.md`-файл.

## Итоги работы

В ходе работы была сформирована компактная коллекция веб-архивов документации по ключевым инструментам современной разработки (12factor, Kubernetes, Prometheus, Docker, FastAPI). Коллекция позволяет:

* на практике отработать полный цикл веб-архивирования: от `wpull` до анализа CLEAR-метрик;
* увидеть, как реальные технические сайты устроены «под капотом» (MIME-типы, внешние ресурсы, ошибки HTML/CSS, политики кэширования и безопасности);
* использовать полученные WARC-архивы и метаданные как исходный материал для учебных и исследовательских задач в области цифровых архивов и DH.

Работу можно продолжать, добавляя новые сайты той же тематики (CI/CD-системы, документация по облачным платформам и т. д.) и расширяя сводную аналитику в `warc-metadata-summary.md`.
